SaaS 回调端

SaaS 回调端接入指南

用户绑定流程中，本服务会主动 POST 到你 SaaS 后端的两个 endpoint。 本文说明这两个 endpoint 的输入输出、回调签名校验机制、4 种语言的最小可跑实现， 以及不部署也能本地试的 curl 自测方法。

本页内容 1. 这两个 URL 是干什么的 2. 回调签名校验 3. 请求字段说明 4. 多语言示例 5. 用 curl 自测

1. 这两个 URL 是干什么的

用户向你接入的微信公众号发送平台账号后，本服务会按下面顺序与你的 SaaS 后端通信：

  1. 本服务收到微信回调，解析出消息中的 OpenID 和用户发的"平台账号"。
  2. POST 到「绑定前确认 URL」，问你"这账号能绑吗"。等你返回 {"allowed": true} 才继续。
  3. 本服务写绑定（或按你拒绝的 reasonCode 拒绝）。
  4. POST 到「绑定结果通知 URL」，把最终结果（绑定成功 / 各种拒绝）告诉 SaaS。
  5. SaaS 在第 4 步收到 outcome: "bound" 后真正发免费额度、标完成、推消息。
绑定前确认 URL

本服务会等这个回调返回再决定是否写绑定。 返 5xx / 超时 / {"allowed": false} 都会让绑定被拒（避免 SaaS 故障时白发额度）。 这里是查你自己用户表的地方。

绑定结果通知 URL

本服务写完绑定（或决定拒绝）后再 POST 通知。 只看 HTTP 2xx，不解析 body，不重试。 这里是发额度 / 标完成的地方。

回调签名密钥

如果在控制台填了它，上面两个回调请求会带 HMAC-SHA256 签名 header； 没填则不带。建议生产环境一定要填，否则任何人能伪造回调。

2. 回调签名校验

只有当你在控制台配置了 回调签名密钥（webhookSecret）时， 两个回调请求才会带下面三个 headers：

Header内容
x-wvb-event account.verify 或 wechat.binding.result
x-wvb-timestamp Unix 秒级时间戳，字符串形式
x-wvb-signature sha256=<hex hmac-sha256>

签名算法：

signing_string = x-wvb-timestamp + "." + 原始请求 body 字节
signature      = HMAC-SHA256(webhookSecret, signing_string)，hex 输出
header         = "sha256=" + signature
三条容易踩的坑
  1. 用原始 body bytes 算签名，不要先 JSON.parse 再 stringify —— 字段顺序、空格不同就会算出不同的签名。
  2. 用常量时间比较（每种语言示例里都演示了）。 直接 === / == 会被时序攻击。
  3. 校验时间戳，5 分钟之外的丢弃，防止被人重放抓到的回调。

参考实现：本服务这一侧的签名生成在 src/lib/tenant_callbacks.js 里， 两边公式必须一致。

3. 请求字段说明

account.verify 请求体

字段类型说明
typestring固定 "account.verify"
tenant_idstring你在控制台配置的租户 ID
platform_account_idstring用户在公众号发送的文本，本服务原样转发——这是用户自己输入的，去你的用户表里查存在性，不要 trust
wechat_open_idstring该公众号下的 OpenID，对该公众号稳定唯一
wechat_official_account_appidstring | null公众号 AppID
wechat_subscribe_statusstring"subscribed" / "unsubscribed" / "unchecked"
message_idstring微信消息 ID，可作幂等 key
received_atstring本服务收到时间，ISO-8601

account.verify 响应约定

SaaS 返回本服务行为
200 + {"allowed": true} 继续走绑定
200 + {"allowed": false, "reasonCode": "..."} 拒绝绑定，reasonCode 进审计 + 透传到结果通知
4xx / 5xx / 超时 视为"无法确认"，拒绝绑定，reason_code = host_account_verification_unavailable。这是故意的失败保险——SaaS 挂了不能白发额度

wechat.binding.result 请求体

字段类型说明
typestring固定 "wechat.binding.result"
tenant_idstring同上
platform_account_idstring涉及的平台账号
outcomestring详见下方 outcome 表
is_boundboolean终态是否绑定成功；发额度只看这个
reason_codestring | null拒绝原因，bound / idempotent_success 时为 null
wechat_binding_refstring | null本服务的绑定记录 ref，可存留备查
wechat_official_account_appidstring | null
wechat_subscribe_statusstring | null
bound_atstring | nullISO-8601
message_idstring微信消息 ID，可作幂等 key
occurred_atstring本服务发出时间，ISO-8601

wechat.binding.result 响应约定

本服务只看 HTTP 2xx；body 不解析。 SaaS 端自己保证幂等—— 同一个 message_id 或 (tenant_id, platform_account_id) 重复收到时只发一次额度。 本服务这次不实现回调重试。

outcome / reason_code 含义

outcomereason_code含义SaaS 应做什么
bound null 第一次成功绑定 发额度
idempotent_success null 同一对 (OpenID, account) 重复绑定 视为已绑；按你 message_id 去重决定是否再发额度
rejected_wechat_already_bound wechat_already_bound 该 OpenID 已绑其他账号 提示用户该微信已用于其他账号
rejected_platform_account_already_bound platform_account_already_bound 该平台账号已绑其他 OpenID 提示用户该账号已绑过微信
rejected_missing_or_expired_intent missing_or_expired_pending_intent 该平台账号没有活跃的 pending intent 引导用户回到任务页重新发起验证
rejected_host_account_verification (SaaS 自己返的) 或 account_not_allowed SaaS 在 verify 阶段说不允许 按你自己返的 reasonCode 处理
rejected_host_account_verification host_account_verification_unavailable SaaS 的 verify 接口挂了或超时 修 SaaS 端，让用户重发消息
rejected_malformed 一般为 missing_platform_account_id 用户发了空消息或非法消息 一般无需处理，本服务已自动回复用户

4. 多语言示例

每段示例都做了同一件事： 校验签名 → 解析 body → 标注"在这里查你自己的用户表"和"在这里发额度"两个扩展点 → 返回响应。 复制出去补上业务逻辑就能用。所有示例都假设环境变量 WVB_WEBHOOK_SECRET 是控制台填的回调签名密钥； 没设置则跳过签名校验（仅本地试用）。 

import express from 'express';
import crypto from 'node:crypto';

const app = express();
const SECRET = process.env.WVB_WEBHOOK_SECRET ?? '';

// 用 verify 拿到原始 body 字节，不要先 JSON.parse 再 stringify
app.use(express.json({
  verify: (req, _res, buf) => { req.rawBody = buf.toString('utf8'); }
}));

function verifySignature(req) {
  if (!SECRET) return true; // 没配密钥时跳过校验

  const ts = req.header('x-wvb-timestamp') ?? '';
  const sigHeader = req.header('x-wvb-signature') ?? '';
  const sig = sigHeader.startsWith('sha256=') ? sigHeader.slice(7) : sigHeader;

  // 5 分钟防回放
  if (Math.abs(Date.now() / 1000 - Number(ts)) > 300) return false;

  const expected = crypto
    .createHmac('sha256', SECRET)
    .update(`${ts}.${req.rawBody ?? ''}`)
    .digest('hex');

  try {
    return crypto.timingSafeEqual(
      Buffer.from(sig, 'hex'),
      Buffer.from(expected, 'hex')
    );
  } catch {
    return false;
  }
}

app.post('/api/wechat/verify-account', (req, res) => {
  if (!verifySignature(req)) return res.status(401).json({ error: 'bad_signature' });

  const { platform_account_id } = req.body;
  // ← 在这里查你自己的用户表，决定是否允许该账号绑定
  const allowed = Boolean(platform_account_id);

  res.json(allowed
    ? { allowed: true }
    : { allowed: false, reasonCode: 'account_not_allowed' });
});

app.post('/api/wechat/result', (req, res) => {
  if (!verifySignature(req)) return res.status(401).json({ error: 'bad_signature' });

  const { outcome, platform_account_id, message_id } = req.body;
  // ← 用 message_id 做幂等：同一个 id 已处理过就跳过
  if (outcome === 'bound') {
    // ← 在这里发免费额度 / 标记完成 / 通知用户
  }

  res.status(200).end();
});

app.listen(8080);
import hashlib
import hmac
import json
import os
import time

from fastapi import FastAPI, HTTPException, Request

app = FastAPI()
SECRET = os.environ.get("WVB_WEBHOOK_SECRET", "")


async def read_verified_body(request: Request) -> dict:
    # 用原始 body 字节算签名，不要先 json.loads 再 dumps
    raw = await request.body()

    if SECRET:
        ts = request.headers.get("x-wvb-timestamp", "")
        sig_header = request.headers.get("x-wvb-signature", "")
        sig = sig_header[7:] if sig_header.startswith("sha256=") else sig_header

        try:
            ts_int = int(ts)
        except ValueError:
            raise HTTPException(401, "bad_timestamp")

        # 5 分钟防回放
        if abs(time.time() - ts_int) > 300:
            raise HTTPException(401, "stale_timestamp")

        expected = hmac.new(
            SECRET.encode(),
            f"{ts}.{raw.decode()}".encode(),
            hashlib.sha256,
        ).hexdigest()

        if not hmac.compare_digest(sig, expected):
            raise HTTPException(401, "bad_signature")

    return json.loads(raw or b"{}")


@app.post("/api/wechat/verify-account")
async def verify_account(request: Request):
    body = await read_verified_body(request)

    # ← 在这里查你自己的用户表，决定是否允许该账号绑定
    if not body.get("platform_account_id"):
        return {"allowed": False, "reasonCode": "account_not_allowed"}
    return {"allowed": True}


@app.post("/api/wechat/result")
async def result(request: Request):
    body = await read_verified_body(request)

    # ← 用 message_id 做幂等；outcome == "bound" 时发额度
    if body.get("outcome") == "bound":
        pass  # ← 在这里发免费额度
    return {"ok": True}
package main

import (
	"bytes"
	"crypto/hmac"
	"crypto/sha256"
	"crypto/subtle"
	"encoding/hex"
	"encoding/json"
	"io"
	"net/http"
	"os"
	"strconv"
	"strings"
	"time"
)

var secret = os.Getenv("WVB_WEBHOOK_SECRET")

// 读 body 并做签名校验。返回原始 bytes 与是否通过。
func verifyAndRead(w http.ResponseWriter, r *http.Request) ([]byte, bool) {
	raw, err := io.ReadAll(r.Body)
	if err != nil {
		http.Error(w, "read_failed", http.StatusBadRequest)
		return nil, false
	}

	if secret == "" {
		return raw, true // 没配密钥时跳过校验
	}

	ts := r.Header.Get("x-wvb-timestamp")
	sig := strings.TrimPrefix(r.Header.Get("x-wvb-signature"), "sha256=")

	tsInt, err := strconv.ParseInt(ts, 10, 64)
	if err != nil || abs(time.Now().Unix()-tsInt) > 300 {
		http.Error(w, "stale_or_bad_timestamp", http.StatusUnauthorized)
		return nil, false
	}

	mac := hmac.New(sha256.New, []byte(secret))
	mac.Write([]byte(ts + "."))
	mac.Write(raw)
	expected := hex.EncodeToString(mac.Sum(nil))

	if subtle.ConstantTimeCompare([]byte(sig), []byte(expected)) != 1 {
		http.Error(w, "bad_signature", http.StatusUnauthorized)
		return nil, false
	}
	return raw, true
}

func abs(n int64) int64 {
	if n < 0 {
		return -n
	}
	return n
}

func handleVerifyAccount(w http.ResponseWriter, r *http.Request) {
	raw, ok := verifyAndRead(w, r)
	if !ok {
		return
	}

	var body map[string]any
	json.NewDecoder(bytes.NewReader(raw)).Decode(&body)
	w.Header().Set("content-type", "application/json")

	// ← 在这里查你自己的用户表，决定是否允许该账号绑定
	if id, ok := body["platform_account_id"].(string); !ok || id == "" {
		json.NewEncoder(w).Encode(map[string]any{
			"allowed":    false,
			"reasonCode": "account_not_allowed",
		})
		return
	}
	json.NewEncoder(w).Encode(map[string]any{"allowed": true})
}

func handleResult(w http.ResponseWriter, r *http.Request) {
	raw, ok := verifyAndRead(w, r)
	if !ok {
		return
	}

	var body map[string]any
	json.NewDecoder(bytes.NewReader(raw)).Decode(&body)

	// ← 用 message_id 做幂等；outcome == "bound" 时发额度
	if body["outcome"] == "bound" {
		// ← 在这里发免费额度
	}
	w.WriteHeader(http.StatusOK)
}

func main() {
	http.HandleFunc("/api/wechat/verify-account", handleVerifyAccount)
	http.HandleFunc("/api/wechat/result", handleResult)
	http.ListenAndServe(":8080", nil)
}

function verifySignature(string $secret, string $raw): bool {
    if ($secret === '') {
        return true; // 没配密钥时跳过校验
    }

    $ts = $_SERVER['HTTP_X_WVB_TIMESTAMP'] ?? '';
    $sigHeader = $_SERVER['HTTP_X_WVB_SIGNATURE'] ?? '';
    $sig = (substr($sigHeader, 0, 7) === 'sha256=')
        ? substr($sigHeader, 7)
        : $sigHeader;

    // 5 分钟防回放
    if (!ctype_digit($ts) || abs(time() - (int)$ts) > 300) {
        return false;

    $expected = hash_hmac('sha256', "$ts.$raw", $secret);
    return hash_equals($expected, $sig); // 常量时间比较
}

5. 用 curl 自测

SaaS 端写完后，不用等微信真的把消息推过来—— 本地用 curl 自己模拟一次本服务的回调请求，能立刻看出签名校验、字段解析、业务分支是否对。

模拟 account.verify

SECRET="your-shared-secret"
TS=$(date +%s)
BODY='{"type":"account.verify","tenant_id":"demo-tenant","platform_account_id":"user_123","wechat_open_id":"o_xxx","wechat_official_account_appid":"wx...","wechat_subscribe_status":"subscribed","message_id":"1001","received_at":"2026-04-25T00:00:00.000Z"}'

SIG=$(printf '%s' "${TS}.${BODY}" | openssl dgst -sha256 -hmac "$SECRET" | awk '{print $2}')

curl -X POST http://localhost:8080/api/wechat/verify-account \
  -H 'content-type: application/json' \
  -H "x-wvb-event: account.verify" \
  -H "x-wvb-timestamp: $TS" \
  -H "x-wvb-signature: sha256=$SIG" \
  --data-raw "$BODY"

模拟 wechat.binding.result（成功绑定）

SECRET="your-shared-secret"
TS=$(date +%s)
BODY='{"type":"wechat.binding.result","tenant_id":"demo-tenant","platform_account_id":"user_123","outcome":"bound","is_bound":true,"reason_code":null,"wechat_binding_ref":"9f4c7a1b2d3e8f00","wechat_official_account_appid":"wx...","wechat_subscribe_status":"subscribed","bound_at":"2026-04-25T00:00:00.000Z","message_id":"1001","occurred_at":"2026-04-25T00:00:00.000Z"}'

SIG=$(printf '%s' "${TS}.${BODY}" | openssl dgst -sha256 -hmac "$SECRET" | awk '{print $2}')

curl -X POST http://localhost:8080/api/wechat/result \
  -H 'content-type: application/json' \
  -H "x-wvb-event: wechat.binding.result" \
  -H "x-wvb-timestamp: $TS" \
  -H "x-wvb-signature: sha256=$SIG" \
  --data-raw "$BODY"
SaaS 端读到的签名校验不通过？

99% 是没用原始 body 字节算签名（先 parse 再 stringify 会破坏空格/字段顺序）； 少数情况是把 timestamp 当成数字传了——header 里始终是字符串。
