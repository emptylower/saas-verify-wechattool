新版微信开发者平台流程

微信公众号接入教程

从微信开发者平台拿到 AppID、AppSecret、API 白名单和消息推送配置， 再回到本项目控制台保存租户配置。关键信息示例均已打码。

配置顺序 1. 进入公众号开发者后台 2. 获取微信侧配置 3. 启用消息推送 4. 回填本项目控制台 5. 端到端验证

1. 进入公众号开发者后台

现在从 微信开发者平台 登录，不再按旧的公众号后台入口找开发配置。

  1. 打开微信开发者平台并登录。
  2. 进入 我的业务与服务。
  3. 在 我的业务 中选择 公众号。
  4. 点击要接入的公众号，进入公众号开发者后台。

2. 获取微信侧配置

进入公众号详情页后，重点看三个区域：基础信息、开发密钥、域名与消息推送配置。

复制 AppID

填入本项目控制台的 微信 AppID。

启用 AppSecret

如果页面显示「重置/冻结」，说明已可用；把 AppSecret 保存到本项目控制台。

启用 API IP 白名单

把运行本服务的出口 IP 加入白名单，否则换取 access_token 会失败。

3. 启用消息推送

这是绑定能不能成功的核心。用户给公众号发消息后，微信只会把消息推送到这里配置的 URL。

  1. 在 域名与消息推送配置 中找到 消息推送。
  2. 点击 启用 或 编辑。
  3. URL 填本服务公网地址：https://你的域名/wechat/{tenantId}/webhook。
  4. Token 建议点微信侧 随机生成，然后复制到本项目控制台。
  5. 消息加密方式选择 明文模式。
  6. 数据格式选择 XML。
  7. 保存后回到列表，确认消息推送处于 启用 状态。
Token 注意事项

Token 必须和本项目控制台里的 微信 Token 完全一致。不要手写带短横线的值； 如果微信提示「请输入正确的 Token」，直接使用微信页面的随机生成值最稳。

4. 回填本项目控制台

微信侧配置完成后，回到本项目 Web 控制台 保存租户配置。

控制台字段 填写来源 说明
Tenant ID 你自己定义 会出现在 webhook URL 中，例如 acme-wechat。
Host Client ID / Secret 你自己定义 SaaS 后端调用本服务 API 时使用，不能暴露给浏览器。
微信 Token 消息推送配置 必须和微信开发者平台消息推送 Token 一致。
微信 AppID / AppSecret 公众号详情页 用于换取 access_token；如果公众号没有 user/info 权限，绑定仍可基于消息 OpenID 完成。
账号校验 URL SaaS 后端 收到公众号消息后，本服务先调用它确认该账号是否允许绑定。
结果通知 URL SaaS 后端 绑定成功或失败后，本服务主动把结果通知给 SaaS。
回调签名密钥 你自己定义 用于给账号校验和结果通知请求生成 HMAC 签名。

5. 端到端验证

  1. SaaS 后端调用 POST /v1/tenants/{tenantId}/pending-bind-intents 创建待绑定会话。
  2. 用户关注公众号，并向公众号发送自己的平台账号。
  3. 本服务收到微信回调，读取 FromUserName 作为该公众号下的唯一 OpenID。
  4. 如果该 OpenID 未绑定其他平台账号，并且平台账号有有效会话，则写入绑定。
  5. SaaS 后端查询 GET /v1/tenants/{tenantId}/bindings/{platformAccountId}，返回 is_bound: true 后再发放免费额度。
验收口径

同一 OpenID 第一次绑定成功；同一 OpenID 再绑定第二个账号时，返回 wechat_already_bound。
